# program-board（ミニマム版プラットフォーム）

探究プログラムを掲示板のように一覧表示し、生徒が参加申込できる検証用 Web アプリ。


# ミニマム版プラットフォーム要件定義

## 1. 概要

- **形態**: Web版。掲示板のように「プログラムが一覧で並び、生徒が参加できる」イメージ。
- **目的**: 検証用に触れるプロダクトを1つ用意する。

---

## 2. アカウントロールと機能

### 管理者

- **掲載状況の閲覧**: 掲載側の掲載状況を閲覧できる。
- **生徒登録状況の閲覧**: 生徒側の登録状況を閲覧できる。
- **生徒アカウントの作成**: 利用する生徒のアカウントを管理者が作成し、配布する形で運用できる。
- **CSV出力**: 生徒の登録情報や掲載状況をCSVで取得できる。

### 掲載側

- **プログラムの登録**: 探究プログラムの登録（作成・公開）ができる。
- **プログラム一覧**: 自分が登録したプログラムの一覧を確認できる。
- **参加者一覧**: 自分が登録したプログラムの参加者（生徒）一覧を確認できる。

### 生徒側

- **プログラム一覧の閲覧**: 公開中の探究プログラム一覧を見られる。
- **プログラム詳細の閲覧**: プログラムの内容・課題・期間等を確認できる。
- **参加申込**: プログラムに「参加する」操作ができる。
- **参加したプログラム一覧**: 自分が参加したプログラムの一覧を確認できる。

---

## 3. プログラムの最小項目

- **ProgramID**: プログラムの管理用ID。
- **プログラム名**: プログラムのタイトル。
- **概要・説明**: プログラムの説明文。
- **課題テーマ**: 探究のテーマ（例：自分らしさや多様性を尊重する社会を作るためには？）。
- **期間・実施形態**: 例）2ヶ月・週1回・全7〜9講。
- **募集状態**: 募集中 / 締切。

---

## 4. 生徒が持つ情報（必須）

- **StudentID**: 管理用ID。
- **メールアドレス**: ログインID兼連絡先。管理者がアカウント作成・配布する際に必要。
- **名前**: ニックネーム。表示・識別用。
- **初期パスワード**: 管理者が設定し、生徒に配布。初回ログイン後に変更できるとよい。

---

## 5. 画面イメージ

- **掲載側**: ログイン → 自分のプログラム一覧 → 登録 / 参加者確認
- **生徒側**: ログイン → プログラム一覧（掲示板）→ 詳細 → 参加する
- **管理者**: ログイン → 掲載状況・生徒登録状況の閲覧 / 生徒アカウント作成・配布
